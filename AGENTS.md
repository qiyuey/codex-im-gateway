# Project Guidelines

## Engineering Quality

- This project is plugin software. Maintain its quality with modern software engineering practices, including unit tests, integration tests, static analysis, and continuous integration where appropriate.
- Add or update automated tests for behavior changes and bug fixes. Verify changes with the relevant test suites before considering work complete.
- Do not use any harness-related skill for this project, including `harness-build`, `harness-fix`, and `harness-review`. Diagnose, implement, review, and validate changes using the project's normal software engineering workflow and tooling instead.
